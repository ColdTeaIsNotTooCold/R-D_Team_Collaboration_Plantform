## 上下文命令

用于管理 `.claude/context/` 中项目上下文的命令。

### `/context:create`
- **用途**：创建初始项目上下文文档
- **用法**：`/context:create`
- **说明**：分析项目结构并在 `.claude/context/` 中创建全面的基准文档。包含项目概览、架构、依赖和模式。
- **使用场景**：项目启动时或需要完全重建上下文时
- **输出**：涵盖项目不同方面的多个上下文文件

### `/context:update`
- **用途**：用最新更改更新现有上下文
- **用法**：`/context:update`
- **说明**：基于最近的代码更改、新功能或架构更新刷新上下文文档。保留现有上下文同时添加新信息。
- **使用场景**：重大更改后或主要工作会话前
- **输出**：带有更改跟踪的已更新上下文文件

### `/context:prime`
- **用途**：将上下文加载到当前对话中
- **用法**：`/context:prime`
- **说明**：读取所有上下文文件并将其加载到当前对话的内存中。保持项目认知的关键。
- **使用场景**：任何工作会话开始时
- **输出**：确认已加载的上下文

## 测试命令

用于测试配置和执行的命令。

### `/testing:prime`
- **用途**：配置测试设置
- **用法**：`/testing:prime`
- **说明**：检测并配置项目的测试框架，创建测试配置，准备测试运行器代理。
- **使用场景**：初始项目设置或测试框架更改时
- **输出**：包含测试命令和模式的 `.claude/testing-config.md`

### `/testing:run`
- **用途**：执行测试并进行智能分析
- **用法**：`/testing:run [test_target]`
- **说明**：使用测试运行器代理执行测试，该代理将输出捕获到日志并仅返回基本结果以保持上下文。
- **选项**：
   - 无参数：运行所有测试
   - 文件路径：运行特定测试文件
   - 模式：运行匹配模式的测试
- **输出**：带有失败分析的测试摘要，主线程中无详细输出

## 实用工具命令

通用实用工具和维护命令。

### `/prompt`
- **用途**：处理包含多个引用的复杂提示
- **用法**：在文件中写入提示，然后输入 `/prompt`
- **说明**：当包含大量 @ 引用的复杂提示在直接输入中失败时使用的临时命令。提示先写入命令文件，然后执行。
- **使用场景**：Claude UI 拒绝复杂提示时
- **输出**：执行写入的提示

### `/re-init`
- **用途**：用 PM 规则更新或创建 CLAUDE.md
- **用法**：`/re-init`
- **说明**：用 `.claude/CLAUDE.md` 中的规则更新项目的 CLAUDE.md 文件，确保 Claude 实例具有正确的指令。
- **使用场景**：克隆 PM 系统或更新规则后
- **输出**：项目根目录中更新的 CLAUDE.md

## 审查命令

用于处理外部代码审查工具的命令。

### `/code-rabbit`
- **用途**：智能处理 CodeRabbit 审查意见
- **用法**：`/code-rabbit` 然后粘贴意见
- **说明**：具有上下文感知地评估 CodeRabbit 建议，接受有效改进同时忽略无上下文感知的建议。为多文件审查生成并行代理。
- **功能**：
   - 理解 CodeRabbit 缺乏完整上下文
   - 接受：真实错误、安全问题、资源泄漏
   - 忽略：风格偏好、无关模式
   - 多文件并行处理
- **输出**：接受/忽略建议的摘要及原因说明

## 命令模式

所有命令都遵循一致的模式：

### 允许的工具
每个命令在前言中指定其所需工具：
- `Read, Write, LS` - 文件操作
- `Bash` - 系统命令
- `Task` - 子代理生成
- `Grep` - 代码搜索

### 错误处理
命令遵循快速失败原则：
- 首先检查先决条件
- 提供解决方案的明确错误消息
- 永不保留部分状态

### 上下文保持
处理大量信息的命令：
- 使用代理保护主线程免受详细输出影响
- 返回摘要而非原始数据
- 仅保留基本信息

## 创建自定义命令

添加新命令：

1. **创建文件**：`commands/category/command-name.md`
2. **添加前言**：
   ```yaml
   ---
   allowed-tools: Read, Write, LS
   ---
   ```
3. **构建内容**：
   - 用途和用法
   - 预检查
   - 分步说明
   - 错误处理
   - 输出格式

4. **遵循模式**：
   - 保持简单（不过度验证）
   - 快速失败并提供明确消息
   - 使用代理进行繁重处理
   - 返回简洁输出

## 与代理集成

命令通常使用代理进行繁重工作：

- **test-runner**：执行测试，分析结果
- **file-analyzer**：总结冗长文件
- **code-analyzer**：在代码库中查找错误
- **parallel-worker**：协调并行执行

这使得在执行复杂工作时保持主对话上下文清洁。

## 说明

- 命令是解释为指令的 markdown 文件
- `/` 前缀触发命令执行
- 命令可以生成代理以保持上下文
- 所有 PM 命令（`/pm:*`）记录在主 README 中
- 命令遵循 `/rules/` 中定义的规则
